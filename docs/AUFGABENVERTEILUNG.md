# Aufgabenverteilung

Dieses Projekt wurde fuer eine Studienabgabe vorbereitet. Die folgende Aufgabenverteilung kann fuer die Dokumentation verwendet und bei Bedarf an die echte Teamstruktur angepasst werden.

| Bereich | Aufgabe | Verantwortlich |
| --- | --- | --- |
| Projektstruktur | Repository, Ordnerstruktur, Abhaengigkeiten | Julian |
| Backend | Server, API-Endpunkte, Hintergrund-Aktualisierung | Julian |
| Datenbeschaffung | Abruf aus URL und Testdaten-Fallback | Julian |
| Data Cleaning | Validierung, Typumwandlung, Zeitformat | Julian |
| Data Storage | SQLite-Datenbank und CRUD-nahe Funktionen | Julian |
| Berechnung | Mittelwert, Tagesenergie, Dashboard-Zusammenfassung | Julian |

| DevOps | Docker, Prometheus, Grafana, GitHub Actions | Julian |
| Dokumentation | README, Projekterklaerung, Dashboard-Spezifikation | Julian |

## Hinweise zur Git-Benutzung

- Entwicklung erfolgt auf dem Branch `main`.
- Aenderungen werden in kleinen thematischen Commits gespeichert.
- Beispiele fuer Commit-Themen:
  - Testdaten hinzufuegen
  - Dashboard-Seite ergaenzen
  - Navigation verbessern
  - CI und Linting einrichten
